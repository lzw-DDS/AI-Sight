AI Sight — 多 Agent 提示词工程工作台
AI 创作者在多模型协作（Midjourney + Kling + Suno 等）中面临提示词管理混乱、版本迭代困难、跨工具提示词适配成本高的痛点。现有工具多为单一模型优化，缺乏系统化的提示词工程工作流。
项目采用三 Agent 协作架构：理解 Agent 负责解析用户创作需求并拆解为结构化任务；生成 Agent 针对每个 AI 工具的特性生成专用提示词，支持多版本迭代优化；执行 Agent 模拟完整工具链调用流程并可视化展示。三个 Agent 通过共享上下文实现长链推理，用户输入一个创作需求，系统自动完成需求理解 → 提示词生成 → 工作流模拟的全链路处理。
项目采用纯前端架构，零 API 成本即可运行，已内置多组典型创作场景的完整输出示例。后续可无缝接入 MiMo API 替换模拟推理，实现真实 AI 驱动。项目可直接演示多 Agent 协作的完整工作流，适合作为 Agent 生态建设的参考案例。

🤖 AI Sight 工作原理

1

理解 Agent

接收用户创作需求，通过结构化分析拆解为可执行任务单元，识别关键词、风格倾向、输出格式要求。

2

生成 Agent

根据任务单元，针对每个 AI 工具（Midjourney / Kling / Suno）的特性，生成专用提示词，支持 A/B 版本对比。

3

执行 Agent

模拟完整工具链调用流程，可视化展示从提示词输入到最终产出的全过程，支持导出全部提示词。

💡 当前版本使用模拟数据展示工作流程。申请通过并获得 MiMo API KEY 后，将替换为真实 AI 推理。
